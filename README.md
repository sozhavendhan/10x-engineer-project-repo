# PromptLab

**Your AI Prompt Engineering Platform**

PromptLab helps AI teams **store, organize, and iterate** on their prompt engineering work. Think of it as a collaborative workspace for managing prompt templates, collections, tags, and version history with the speed of a Postman-style workflow.

---


## Overview


PromptLab exposes a FastAPI backend that lets you manage prompts and collections entirely in memory today. Over the next few weeks you will:


- Fix bugs and stabilize the API
- Add comprehensive documentation and specs
- Harden the backend with tests, CI, and Docker
- Build a React frontend to manage prompts visually


This README is your single source of truth for onboarding, setup, and API references.






---


## Key Features


- **Prompt Templates**: Capture prompt text, descriptions, and optional collection associations.
- **Collections**: Group prompts by project, persona, or workflow.
- **Query Support**: Filter prompts by collection or keyword search with built-in utilities.
- **API First**: FastAPI routes with autogenerated OpenAPI docs at `/docs`.
- **In-Memory Storage**: Simple storage backend that can be swapped for a real database in Week 3.
- **Spec-Ready**: Architecture built for upcoming features like version tracking and tagging.

## Getting Started

### Prerequisites

- Python 3.10+

- Git
- (Optional) Node.js 18+ for upcoming frontend work


### Installation & Setup





1. Clone the repository and switch to the backend folder.
2. Set up a virtual environment and install dependencies.
3. Launch the FastAPI application with Uvicorn.



```bash README.md
git clone <your-repo-url>
cd promptlab/backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

uvicorn app.api:app --reload --host 0.0.0.0 --port 8000
```


The API will be available at `http://localhost:8000` and the interactive docs at `http://localhost:8000/docs`.


### Running Tests




```bash README.md
cd backend
source .venv/bin/activate
pytest tests/ -v
```

## API Summary

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Check service health and version |
| GET | `/prompts` | List prompts with sorting, filtering, and search |
| GET | `/prompts/{id}` | Retrieve a prompt by ID |
| POST | `/prompts` | Create a new prompt template |
| PUT | `/prompts/{id}` | Replace an existing prompt |
| PATCH | `/prompts/{id}` | Partially update a prompt field-by-field |
| DELETE | `/prompts/{id}` | Remove a prompt (hard delete) |
| GET | `/collections` | List all collections |
| GET | `/collections/{id}` | Retrieve a single collection |
| POST | `/collections` | Create a collection |
| DELETE | `/collections/{id}` | Delete a collection and orphan its prompts |

## Usage Examples

### Create a Prompt

```bash README.md
curl -X POST http://localhost:8000/prompts \
  -H 'Content-Type: application/json' \
  -d '{"title": "Summarize Support Ticket", "content": "Write a concise summary for {{input}}.", "description": "Summarizes user support tickets for internal review.", "collection_id": null}'
```
























### List Prompts with Filters

```bash README.md
curl "http://localhost:8000/prompts?search=support&collection_id=12345"
```


The response includes metadata and prompts sorted with newest first.


### Partially Update a Prompt






```bash README.md
curl -X PATCH http://localhost:8000/prompts/{prompt_id} \
  -H 'Content-Type: application/json' \
  -d '{"description": "Sharper summary tone"}'
```


### Delete a Collection Safely





```bash README.md
curl -X DELETE http://localhost:8000/collections/{collection_id}
```
